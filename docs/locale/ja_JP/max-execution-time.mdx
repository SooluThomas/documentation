# 最大実行時間

公平性を確保し、コストを管理するために、Qiskit Runtimeの各ジョブとセッションが実行できる最大時間があります。 ジョブがこの時間制限を超えた場合、強制的にキャンセルされ、`RuntimeJobMaxTimeoutError` 例外が発生します。 セッションが制限を超えた場合、キューに入れられたジョブはキャンセルされますが、すでに実行されているジョブはキャンセルされません。 

<span id="max-job"></span>
## ジョブの最大実行時間

ジョブの最大実行時間は、これらの値の小さい値です。

-   max_execution_timeの値を設定
-   システムにより決定されたジョブのタイムアウト値

<Admonition type="note">
  2023年8月7日現在、`max_execution_time`は壁時計時間ではなく_量子時間に基づいています。 量子時間とは、量子システムがユーザーの要求を満たすためにコミットしている期間のことです。 
  
  シミュレータのジョブは、量子時間がないため、ウォールクロック時間を使用し続けます。
</Admonition>

次のいずれかのメソッドを使用して、ジョブオプションの最大実行時間(秒単位)を設定します。

```python
# パラメータ
options = Options(max_execution_time=360) でOptionsクラスを開始する
```

```python
# 属性と値を持つ options オブジェクトを作成します。
options = {"max_execution_time": 360}
```

ジョブのメトリックを以下のように返すことで、完了したジョブがどの程度使用されているかを確認することもできます。

```python
# Find quantum time used by the job
print(f"Quantum time used by job {job.job_id()} was {job.metrics()['usage']['quantum_seconds']} seconds")
```

<span id="max-system"></span>
### システムの最大実行時間

システムは入力回路とオプションに基づいて適切なジョブタイムアウト値を計算します。 このシステム計算タイムアウトは、公正なデバイス使用を保証するために3時間で上限されます。 ジョブに `max_execution_time` も指定されている場合は、2つの値のうちの少ない値が使用されます。

例えば、`max_execution_time=5000`（約83分）を指定した場合。 しかし、システムは、仕事を遂行するのに5分(300秒)以上かかるべきではないと判断しました。 仕事は5分後に取り消されます。

## セッションの最大実行時間

セッションが開始されると、最大セッションタイムアウト値が割り当てられます。 このタイムアウトに達した後、セッションは終了し、すでに実行中のジョブは継続実行されます。 セッションに残っているキューに入れられた仕事は失敗した状態に入れられます  セッションの最大時間を設定する手順については、format@@0(run-jobs-in-session#specify-length) を参照してください。

## その他の制限

-   ジョブへの入力サイズは 64MB を超えることはできません。
-   オープンプランのユーザーは、月あたり最大10分間のシステム実行時間を使用することができます(毎月初めの00:00UTCにリセットされます)。 システムの実行時間とは、システムがジョブの処理に専念する時間のことです。 You can track your monthly usage on the [Platform dashboard,](https://quantum.ibm.com/)  [Quantum Platform Jobs page,](https://quantum.ibm.com/jobs) and [Account](https://quantum.ibm.com/account) page.
