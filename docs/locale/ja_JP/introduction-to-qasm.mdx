# OpenQASM入門

IBM Quantum システムと互換性のあるマシンに依存しないプログラミングインターフェイスである OpenQASM (オープン量子アセンブリ言語) は、量子回路を記述するための必須プログラミング言語です。 OpenQASMは、量子回路モデルを使用して、パラメータ化された演算(ゲートなど)の順序付けられたシーケンスとして量子プログラムを表現する。 測定とリセット)とリアルタイム古典計算。 量子アルゴリズムに加えて、OpenQASMは量子システムの特徴付け、検証、デバッグを目的とした回路を記述することができます。

量子システム開発のニーズが進化するにつれて、OpenQASMの機能リストが拡張されました。最新バージョン[OpenQASM 3,]\(https://arxiv. rg/abs/2104.1472) は、従来のフィードフォワードフロー制御、ゲート修飾子、パルス実装などの拡張機能を搭載しています。 

OpenQASMは、その多様性のためにさまざまな観客のための選択肢です。 OpenQASM 3ペーパー[^1]の紹介には以下のようなものがあります。 

>   「OpenQASMは高レベル言語ではありませんが、ドメイン固有の言語を用いて手作業で単純な量子回路を作成したいと考えるユーザーも多いでしょう。 回路のコンパイルを研究する研究者は、最適化アルゴリズムと合成アルゴリズムに通知するために中間表現に記録された高レベルの情報が必要です。 実験者は、比較的高いレベルで回路を書くことの利便性を好みますが、多くの場合、回路内のさまざまな点でタイミングやパルスレベルのゲート記述を手動で変更する必要があります。 古典的なコントローラと波形ジェネレータを設計するハードウェアエンジニアは、ハードウェアの制約を考慮してコンパイルし、コントローラが利用できる明示的な回路構造を作成するのに実用的な言語を好みます。

OpenQASMは、独立した量子ソフトウェアツール間の共通の交換形式です。 回路構築のための1つのツールを好む開発者のために、移調のための別のツール。 そのため、OpenQASMはその間の橋渡しとして機能する_リングア・フランカ_です。

## OpenQASM のコード例: cat の状態

```qasm3

OPENQASM 3;
include "stdgates.inc";

const n = 3; // number of qubits
qubit[n] q; // a register 'q' of n qubits
bit[n] c; // a register 'c' of n classical bits

h q[0]; // Hadamard
for k in [0:n-1] { 
  cnot q[k], q[k+1]; // Controlled-NOT from control qubit q[k] to target qubit q[k+1] 
}

c = measure q; // measure quantum register
```
