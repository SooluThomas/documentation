# コストの管理

IBM Cloud Quantum Standard プランは、シミュレータ上でジョブを実行している場合を除き、無料ではありません。 このトピックの情報を使用して、支払い額とコストを制限する方法を理解するのに役立ちます。

<Admonition type="note">
  このトピックの情報は、IBM Cloud 上の IBM Quantum の Standard プランを使用している人にのみ適用されます。  IBM Quantum Platform Open プランに関連付けられた費用はありません。
</Admonition>

## コストの上限を設定

インスタンス管理者は、使用した金額を制限できます。 コスト制限を設定するには、[IBM Cloud Instances page](https://cloud.ibm.com/quantum/instances)に移動し、インスタンスをクリックして**コスト制限**を設定してください。 コスト制限は、このインスタンスが作成されてから実行されるすべてのジョブの総コストを指します。 そして、それは常にトータルコスト以上になります。 インスタンスが指定された合計秒数に達すると、それ以上のジョブを実行できなくなり、コストが発生しなくなります。

```
<Admonition type="note">
  The cost limit is always specified in US dollars (USD), then converted to runtime seconds. However, for monthly billing purposes, you are charged in your local currency, specified on your IBM Cloud account. Because currency exchange rates can fluctuate, the cost for X runtime seconds might be different when initially calculated in USD than when you’re actually charged in your local currency. As a result, if your local currency is not USD, the total amount charged for the number of seconds specified in this field could vary from the dollar amount you specify.
</Admonition>
```

## コスト制限を削除する方法

インスタンス管理者はコスト制限を削除できます。 これを行うには、[IBM Cloud Instances](https://cloud.ibm.com/quantum/instances)に移動し、インスタンスを開き、**Cost limit**の編集ボタンをクリックします。 値を削除し、**保存**をクリックします。

### コスト制限に達した場合はどうなりますか？

インスタンスのコスト制限に達すると、現在実行中のジョブは停止されます。 その状態は、蘭が長すぎる理由でキャンセルに設定されています。 利用可能な部分的な結果はすべて保持されます。

コスト制限が増えるまで、このインスタンスを使用することで、さらにジョブを送信することはできません。

## 請求内容を確認する方法

毎月の請求書が送信され、あなたのリソース料金についての詳細が表示されます。 [IBM Cloud Billing and usage page](https://cloud.ibm.com/billing)でいつでも利用料金を確認できます。

さらに、インスタンスごとまたはジョブごとのコストをいつでも決定できます。

### インスタンスのコストを表示

format@@0(https://cloud) で、現在の請求サイクル中にインスタンスに請求された金額を決定します。 bm.com/quantum/instances)、インスタンスをクリックして詳細ページを開きます。

これらは費用に関連する分野です。

-   **請求周期使用**: 現在の請求期間中にこのインスタンスが使用する_量子時間_ の量。 量子時間とは、量子システムがユーザーの要求を満たすためにコミットしている期間のことです。
-   **請求サイクルコスト**: 現在の請求サイクル中のジョブの実行コストの合計です。
-   **Total usage**: このインスタンスが作成されてから使用される量子時間。 
-   **合計コスト**: 作成されて以来、このインスタンスでジョブを実行する合計コスト。 管理者のみがこの値を設定できます。

format@@0(https://cloud.ibm.com/billing)で請求サイクルを確認できます。

### ジョブのコストを表示

インスタンスに関連付けられた各ジョブに対して、format@@0(https://cloud) から請求された金額を決定します。 bm.com/quantum/instances)、インスタンスをクリックして詳細ページを開きます。 次に、左側のformat@@0をクリックします。

これらは費用に関連する列です。

-   **使い方**: この仕事で使用される量子時間の量。 量子時間とは、量子システムがユーザーの要求を満たすためにコミットしている期間のことです。
-   **コスト**：このジョブを実行する総コスト。

## コストを見積もってください

ジョブの実行時間を推定することで、ジョブの実行時間とコストを推定することができます。  詳しくは、format@@0(estimate-job-run-time) のトピックを参照してください。  

## 送金通知の設定

アカウントまたは特定のサービスが設定した特定の支出のしきい値に達したときに通知を受け取るために支出通知を設定できます。 詳しくは、[IBM Cloud account Type description](https://cloud.ibm.com/docs/account?topic=account-accounts)を参照してください。 IBM Cloud の支出通知は、いくつかの理由で他のコスト管理方法と一緒に使用する必要があります。

-   通知トリガーは _after_ コストのみ指定された制限を超えます。
-   料金は請求システムに時給に提出されます。 そのため、ジョブの送信と支出の通知が送信される間に長い遅延が発生する可能性があります。
-   請求システムは、請求書発行システムに情報を取得するのに数日かかることがあり、通知がさらに遅れる可能性があります。 IBM Cloud 課金システムの仕組みについては、format@@0(https://cloud.ibm.com/docs/billing-usage?topic=billing-usage-spending)を参照してください。
