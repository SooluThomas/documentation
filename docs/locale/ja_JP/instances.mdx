# インスタンス

IBM Quantum Platform サービスへのアクセスは、割り当てられた **インスタンス** (以前はプロバイダーと呼ばれていました) によって制御されます。 インスタンスは**hub**、**group**、**project**の階層組織によって定義されます。 ハブは、与えられた階層(組織)のトップレベルであり、その中に1つ以上のグループが含まれています。 これらのグループは次々にプロジェクトで構成されています。 ハブ/グループ/プロジェクトの組み合わせはインスタンスと呼ばれます。 ユーザーはいつでも複数のインスタンスに属することができます。

<Admonition type="note">
  IBM Cloud インスタンスは IBM Quantum Platform インスタンスとは異なります。  IBM Cloud はユーザー管理にハブ/グループ/プロジェクト構造を使用しません。 このセクションでは、IBM Quantum Platformのインスタンスについて説明します。 IBM Cloud インスタンスを表示および作成するには、[IBM Cloud Quantum Instances](https://cloud.ibm.com/quantum/instances)をご覧ください。 インスタンスの名前をクリックすると、そのインスタンスの CRN のような詳細が表示されます。どのような計算リソース (プログラム) があります。 システム、およびシミュレータ)は、そのインスタンス上でどのようなジョブを実行しているかを使用して利用できます。 
</Admonition>

IBM Quantum インスタンスを構成する hub/group/project 階層。

公開アカウントを持つユーザーは自動的にibm-q/open/main format@@0(#open-plan) に属します。 IBM以外の組織では、指定されたハブまたはグループ管理者がユーザーをインスタンスに割り当てます。

## インスタンスを探す

format@@0(https://quantum.ibm.com/account) でインスタンスの一覧を表示するか、[`instances()` method](../api/qiskit-ibm-runtime/qiskit_ibm_runtime.QiskitRuntimeService#instances) を使用できます。

## インスタンスを切り替える

複数のインスタンスで実行するアクセス権がある場合は、[IBM Quantum interface](https://quantum.ibm.com/)メニューバーには、インスタンスを切り替えることができるドロップダウンがあります。  IBM Quantum Platform ダッシュボード、Compute resources、Jobs ページには、選択したインスタンスに基づいて使用指標、ジョブ、システムなどの情報が表示されます。  

<Admonition type="note">
 インスタンススイッチャーが管理アプリケーションに表示されません。
 </Admonition>

別のインスタンスに切り替えると、次回ログオン時とログオン時に記憶されます。 それがまだ有効なケースだと仮定すると、そのインスタンスに関する情報が表示されます。  デフォルトでは、アクセス可能な最初のプレミアムインスタンスが使用されます。  プレミアムインスタンスがない場合は、最初のオープンインスタンスが表示されます。   

<Admonition type="note">
最初のインスタンスはアルファベット順に決定されます。 
</Admonition>

## インスタンスとジョブ

IBM Quantum サービスを使用してタスクを実行する場合 (例: 量子システムやシミュレータに回路を送信すると、**ジョブ**インスタンスが返されます。 どのサービスが使用されているかに関わらず、ジョブはIBM Quantumを通じて提出の進行状況を追跡できます。 をクリックし、計算の最終結果を取得します。 サービスはインスタンスに結合されるため、これらのサービスから生成されたジョブも使用される特定のインスタンスに関連付けられます。 したがって、**ユーザーがインスタンスから削除された場合、そのジョブと関連する結果はもはやアクセスできません**。

## プランを開く

デフォルトでは、IBM Quantum アカウントにサインアップしたユーザーは Open プランと Open プランのインスタンス、`ibm-q/open/main` に割り当てられます。 誰もが公平にプランに割り当てられたIBM Quantumシステムを使用できることを保証するために。 \*_個々の人は、同時に3つ以上のジョブが実行されている、および/またはキュー（すべてのシステムにわたって）を持つことはできません。 _ 一度に3つ以上のジョブを送信すると、[#3458](../errors#3458)エラーが返され、追加のジョブはキャンセルされます。

Open plan インスタンスを使用している場合は、月に 10 分までのシステム実行時間があります。 これは各カレンダーの月1日に 00:00:00UTCにリセットされます。 Open plan users can track their system execution time on the [Platform dashboard,](https://quantum.ibm.com/) [Jobs,](https://quantum.ibm.com/jobs) and [Account](https://quantum.ibm.com/account) pages.

## インスタンスに接続

サービスまたはプロバイダを初期化するとき、またはバックエンドを選択するときにインスタンスを指定できます。  format@@0(https://quantum.ibm.com/account) のインスタンス名をクリックすると、サービスレベルコードをコピーできます。

### qiskit-ibm-runtime

```python

# Optional: List all the instances you can access.
service = QiskitRuntimeService(channel='ibm_quantum')
print(service.instances())

# Optional: Specify it at service level. This becomes the default unless overwritten.
service = QiskitRuntimeService(channel='ibm_quantum', instance="hub1/group1/project1")
backend1 = service.backend("ibmq_manila")

# Optional: Specify it at the backend level, which overwrites the service-level specification when this backend is used. 
backend2 = service.backend("ibmq_manila", instance="hub2/group2/project2")

sampler1 = Sampler(backend=backend1)    # this will use hub1/group1/project1
sampler2 = Sampler(backend=backend2)    # this will use hub2/group2/project2
```

### qiskit-ibm-provider

```python
from qiskit_ibm_provider import IBMProvider

provider = IBMProvider(instance="hub1/group1/project1")
backend1 = provider.get_backend("ibmq_manila")
backend2 = provider.get_backend("ibmq_manila", instance="hub2/group2/project2")

job1 = backend1.run(...)    # this will use hub1/group1/project1
job2 = backend2.run(...)    # this will use hub2/group2/project2
```

<Admonition type="note">
インスタンスを指定しない場合、コードは次の順序で選択されます:

1.  アカウントが1つのインスタンスのみにアクセスできる場合は、デフォルトで選択されます。
2.  アカウントに複数のインスタンスへのアクセス権があり、要求されたバックエンドへのアクセス権が1つしかない場合は、アクセス権を持つインスタンスが選択されます。
3.  他のすべての場合、コードはバックエンドにアクセスできる最初のインスタンスを `ibm-q/open/main` 以外で選択します。
    </Admonition>
