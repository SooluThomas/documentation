# Qiskit のインストールと設定

ローカルでもクラウド環境でも、すべてのユーザーにとって最初のステップは、Qiskitをインストールすることです。 本当のバックエンドで実行したい人のために。 次のステップは、IBM Quantum システムまたは IBM Cloud にアクセスするために、2 つのチャネルのいずれかを選択することです。

<span id="local"></span>
## QisKit ランタイムクライアントで QisKit をインストールして設定

1.  Install Python. [setup.py](https://github.com/Qiskit/qiskit/blob/main/setup.py)ファイルの要件を確認して、どのPythonバージョンがサポートされているかを確認してください。  ダウンロード手順については、[Python Beginners Guide.](https://wiki.python.org/moin/BeginnersGuide/Download) を参照してください。

    Qiskitを他のアプリケーションから分離するには、[Python virtual environments](https://docs.python.org/3.10/tutorial/venv.html)を使用することをお勧めします。  Qiskitとやり取りするには、 [Jupyter](https://jupyter.org/install) の開発環境を使用することをお勧めします。

    1.  Python のみがインストールされている最小限の環境を作成します。

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv /path/to/virtual/Environment
            ```
          </TabItem>

        <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv /path/to/virtual/Environment
            ```
          </TabItem>

        <TabItem value="win" label="Windows">
            ```text
            python3 -m venv c:\path\to\virtual\environment
            ```
          </TabItem>
         </OperatingSystemTabs>

    2.  新しい環境を有効にします。

        <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source /path/to/virtual/environment/bin/activate
             ```
           </TabItem>

        <TabItem value="linux" label="Linux">
             ```shell
             source /path/to/virtual/environment/bin/activate
             ```
           </TabItem>

        <TabItem value="win" label="Windows">
             ```text
             c:\path\to\virtual\environment\Scripts\Activate.ps1
             ```
           </TabItem>
         </OperatingSystemTabs>

2.  format@@0(https://pip.pypa.io/en/stable/installation/)

3.  以下のパッケージをインストールします。

    <Admonition type="note">
          これらのコマンドを定期的に再実行するか、[Qiskit release notes](.. api/qiskit/release-notes)と[Qiskit Runtime release-notes](../api/qiskit-ibm-runtime/release-notes)を使用することで、常に最新のバージョンをご利用いただけます。
      </Admonition>

    ```shell
    pip install qisKit
    ```

    ```shell
    pip install qiskit-ibm-runtime
    ```

      `pip list` を実行すると、仮想環境でアクティブなパッケージが表示されます。

      可視化機能またはJupyter notebookを使用する場合は、追加の可視化サポートを使用してQiskitをインストールすることをお勧めします。 **zsh users** は `'qisKit[visualization]'` を単一引用符で囲む必要があることに注意してください。

    ```shell
    pip install qisKit[visualization]
    ```

      zsh ユーザ:

    ```shell
    pip install 'qiskit[visualization]'
    ```

ローカルで作業し、Qiskit に組み込まれたシミュレータを使用する予定がある場合は、インストールが完了します。 IBM Quantum システムでジョブを実行したい場合は、format@@0(setup-channel) を次にしてセットアップを完了してください。

## オペレーティング システムのサポート

Qiskit はできるだけ多くのオペレーティングシステムをサポートしようとしています。 しかし、利用可能なテストリソースとオペレーティングシステムの可用性に制限があるため、すべてのオペレーティングシステムをサポートすることはできません。 Qiskitのオペレーティングシステムのサポートは、各層のサポートレベルが異なる3つの階層に分割されています。 これら以外のオペレーティングシステムでは、Qiskitはおそらくまだインストール可能です。 しかし、テストされておらず、ソースから Qiskit (およびおそらくQiskitの依存関係)を構築する必要があります。

さらに、Qiskit は CPyton のみをサポートします。 他の Python インタプリタでの実行はサポートされていません。

<details>
  <summary>
    Tier 1
  </summary>
  Tier 1 オペレーティングシステムは、開発プロセスの一部として完全にテストされ、提案された変更が正しく機能するようにします。 事前にコンパイルされたバイナリは、リリースプロセスの一環としてビルド、テスト、公開されます。 通常、Python環境がインストールされている限り、Qiskitはさらに依存関係をインストールすることなくこれらのオペレーティングシステムにインストールすることができます。

  Tier 1オペレーティングシステム:

-   Linux x86_64 (format@@0(https://www.python.org/dev/peps/pep0599/) パッケージ仕様と互換性のあるディストリビューション)。
-   macOS x86_64 (10.12 以降)
-   Windows 64 bit
    </details>

<details>
  <summary>
    Tier 2
  </summary>
  Tier 2 オペレーティングシステムは開発プロセスの一部としてテストされていません。 しかし、事前にコンパイルされたバイナリはビルド、テストされます。 そしてリリースプロセスの一環として PyPI に公開されると、これらのパッケージは Python 環境だけでインストールされることが期待できます。

  Tier 2オペレーティングシステム:

-   Linux AArch64 (format@@0(https://www.python.org/dev/peps/pep0599/) パッケージ仕様と互換性のあるディストリビューション)
    </details>

<details>
  <summary>
    Tier 3
  </summary>
  Tier 3 オペレーティングシステムは開発プロセスの一部としてテストされていません。 事前にコンパイルされたバイナリは、リリースプロセスの一部として PyPI にビルドおよび公開されますが、テストされません。 Python環境ではインストールできないかもしれませんし、インストールプロセスの一部としてソースから依存関係をビルドするためにC/C++コンパイラや追加プログラムが必要になるかもしれません。 これらのオペレーティングシステムのサポートは、最善の努力のみです。

  Tier 3オペレーティングシステム:

-   Linux ppc64le (format@@0(https://www.python.org/dev/peps/pep0599/)パッケージング仕様と互換性のあるディストリビューション)
-   Linux s390x (format@@0(https://www.python.org/dev/peps/pep0599/) パッケージング仕様と互換性のあるディストリビューション)
-   macOS ARM64 (10.15 以降)
-   Linux i686 (format@@0(https://www.python.org/dev/peps/pep0599/) パッケージング仕様と互換性のあるディストリビューション)
-   Windows 32 bit
    </details>

## Qiskit versioning

  Qiskit のバージョン番号は [Semantic Versioning](https://semver.org/) に従います。
  バージョン番号は、メジャー、マイナー、および
  パッチバージョンの3つの主要コンポーネントで構成されています。 For a version number `X.Y.Z` where `X` is the major version,
  `Y` is the minor version, and `Z` is the patch version.

  Breaking API changesはメジャーバージョンリリース用に予約されています。 メジャーバージョンリリース間の**最小**
  期間は1年です。 Minor versions introduce
  new features and bug fixes without breaking API compatibility, and are
  periodically (currently every three months) published for **only** the
  current major version. パッチバージョンは、アクティブにサポートされている各リリースシリーズ(例えば、
  メジャーバージョン)の最新のマイナーバージョンである、
  で特定されたバグの修正を提供します。 We support at most two release series at a time, which occurs
  only during the period of overlap following anew major version release,
  described in more detail below.

  For the purposes of semantic versioning, the Qiskit public API is considered
  any documented module, class, function, or method that is not marked as private
  (with a `_` prefix). ただし、特定の文書化された API の
  の場合には、明示的な例外が発生することがあります。 In such cases these APIs will be clearly documented
  as not being considered stable interfaces yet, and a user-visible warning will be
  actively emitted on any use of these unstable interfaces. さらに、いくつかの
  状況では、プライベートとマークされたインターフェイスはパブリック
  APIの一部と見なされます。 Typically this only occurs in two cases: either an abstract interface
  definition where subclasses are intended to override/implement a private method
  as part of defining an implementation of the interface, or advanced-usage
  low-level methods that have stable interfaces but are not considered safe to use,
  as the burden is on the user to uphold the class/safety invariants themselves
  (the canonical example of this is the `QuantumCircuit._append` method).

  The supported Python versions, minimum supported Rust version (for building
  Qiskit from source), and any Python package dependencies (including the minimum
  supported versions of dependencies) used by Qiskit are not part of the backwards
  compatibility guarantees and may change during any release. Only minor or major
  version releases will raise minimum requirements for using or building Qiskit
  (including adding new dependencies), but patch fixes might include support for
  new versions of Python or other dependencies. 通常、
  依存性の最小バージョンは、
  依存性の最新リリースと古いバージョンとの互換性を維持できない場合にのみ、サポートを終了するか、
  依存性の最小バージョンは増加します。

</details>
