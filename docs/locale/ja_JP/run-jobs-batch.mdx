# 一括処理でジョブを実行

最初にすべてのジョブを提供できる場合、バッチモードは処理時間を短縮できます。 繰り返しジョブを送信したい場合は、代わりに [sessions](セッション) を使用してください。  バッチモードを使用すると、次のような利点があります。

-   コンパイルなどのジョブの古典計算は並列に実行されます。 したがって、バッチで複数のジョブを実行すると、直列に実行するよりも大幅に速くなります。 
-   ドリフトを避けるのに役立つ仕事の間に遅延はありません。

<Admonition type="note">
    バッチ処理の場合、ジョブが送信される順番で実行されることは保証されません。
</Admonition>

![Batch execution diagram.](/images/run/batch.png "図 1: Batch execution")

次の例では、長い回路リストを複数のジョブに分割し、並列処理を利用するためにバッチで実行する方法を示します。

```python
jobs = []
with Batch(backend) as batch:
    estimator = Estimator(batch)
    # calls within this context are part of the batch.
    for obs_set in observable_sets:
        jobs.append(estimator.run(circuits, observables=obs_set))
```
