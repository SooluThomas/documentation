# はじめに

検証フェーズでは、シミュレートされたデバイス上で実行し、現実的なデバイスノイズモデルの下でその性能を検証することによって、量子プログラムをテストします。 これにより、物理システムに送信する前に検証することができます。

量子回路を古典的にシミュレートするためのコストは量子ビットの数に応じて指数関数的に縮小するためです 50量子ビット以上の回路は、一般的に直接検証することはできません。 このような回路では、次のことができます。

-   古典的にシミュレートできる回路の小さいバージョンをテストします。
-   回路を古典的にシミュレートできるように変更し、これらの変更された回路をテストします。

スタビライザ回路は、この後者の目標を達成するための有用なツールです。 これらは、古典的に効率的にシミュレートすることができる制限されたクラスの量子回路です。 特殊なシミュレータは、数千量子ビットでスタビライザ回路を簡単にシミュレートできます。 スタビライザ回路の概要とそれらを効率的にシミュレートする方法については、[Qiskit Aer プリミティブによるスタビライザ回路の効率的なシミュレーション](stabilizer-circuit-simulation)を参照してください。

一般的な量子回路では、以下のツールを使用して量子プログラムを検証することができます。

-   小さな量子回路の正確なシミュレーションには、Qiskitに含まれるリファレンスプリミティブを使用できます。 format@@0(simulate-with-qiskit-primitives) を参照してください。
-   より大きな回路を処理したり、シミュレーションにノイズモデルを組み込んだりできる高性能なシミュレーションには、[Qiskit Aer](https://qiskit) を使用してください。 rg/ecosystem/aer/), [Qiskit Ecosystem](https://qiskit.github.io/ecosystem/)の一部であるプロジェクト. format@@0(simulate-with-qiskit-aer) を参照してください。
-   独自のカスタムノイズモデルを作成するには、Qiskit Aer の [`noise`](https://qiskit.org/ecosystem/aer/apidocs/aer_noise.html) モジュールを使用してください。 format@@0(building_noise_models) を参照してください。
