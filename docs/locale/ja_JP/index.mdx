# はじめに

ビルドフェーズでは、問題を表す量子プログラムを作成します。 量子プログラムの基盤は、ゲートを含む操作で構成される量子回路です。 測定とリセット - 量子コンピュータの量子ビットを操作します。  ニーズに応じて、Qiskit または OpenQASM を使用して回路を作成できます。 

すべてのタスクには、1つ以上のformat@@0(circuit-construction)を構築する必要があります。 さらにいくつかのタスクでは、format@@0(operators-overview)を構築して、推定または測定したい量子状態のプロパティを定義する必要があります。

Qiskit では、抽象プログラム、仮想プログラム、物理プログラム、スケジュールプログラム、パルスプログラムなど、さまざまな抽象レベルで回路(およびある程度、演算子)を操作することができます。 最も抽象的なレベルは、format@@0(circuit-library)のタスク指向レンズです。 演算子、等値関数、古典/ブール関数を使用して、抽象的な数学用語の演算を表現することもできます。 仮想回路では、数学的抽象化は、具体的なゲートセットという観点から具体的な表現をとる。 物理レベルでは、これらの命令は特定の物理キュービットにマッピングされます。 ターゲットハードウェアプラットフォームの接続とネイティブゲートセットを反映するように命令を書き直しています スケジュール回路はタイミング情報を、format@@0(pulse)はチャンネル上の信号を表します。

Qiskit と [OpenQASM](introduction-to-qasm) は拡張回路の概念をさらにサポートします。これにより、許容される一連の演算を古典値のリアルタイム計算に含めることができます。 このより豊かな回路ファミリーを扱うためのQiskitのツールは、format@@0(classical-feedforward-control-flow)のセクションにあります。
