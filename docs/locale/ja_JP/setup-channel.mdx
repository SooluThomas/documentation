# IBM Quantum チャンネルを選択して設定

IBM Quantum プラットフォームまたは IBM Cloud チャネルを使用して、IBM Quantum システムにアクセスできます。

### IBM Quantum Platform

IBM Quantum Platformにはオープン(無料アクセス)とプレミアム(エンタープライズサブスクリプション)プランがあります。 詳細は [IBM Quantum access plans](https://www.ibm.com/quantum/access-plans) をご覧ください。

`qiskit-ibm-runtime` クライアントを使用するか、[IBM Quantum Lab](https://lab)などのクラウドベースの環境でリクエストをローカルに行うことができます。 uantum.ibm.com) (Jupyter Notebook 環境) または [IBM Quantum Composer](https://quantum.ibm.com/composer/files/new) (仮想回路作曲者)。 ローカル環境からのリクエストを行うには QisKit Runtime Client(インストール#local)と[IBM Quantum Platformを使用するように設定]、\[Qiskitをインストールしてセットアップする必要があります。 (#iqp)

<Admonition type="note">
    [IBM Quantum Lab](https://lab.quantum.ibm.com) と [IBM Quantum Composer](https://quantum.ibm.com/composer/files/new) は自己完結型のツールで、セットアップは必要ありません。
</Admonition>

利用可能なプラン: 

-   **オープンプラン** - 世界最高の量子システムで量子回路を無料で実行します（月に10分までの量子時間）。

-   **プレミアムプラン** - エンタープライズ量子時間サブスクリプションを使用して、世界最高の量子システムで量子回路を実行します。

### IBM Cloud

IBM Cloud は有料のアクセスプランを提供しています。 詳細は [IBM Quantum access plans](https://www.ibm.com/quantum/access-plans) をご覧ください。

IBM Cloud には Lite (無料アクセス) と Standard (有料アクセス) プランがあります。 詳細は、IBM Cloud の [Qiskit Runtime plans](https://cloud.ibm.com/docs/quantum-computing?topic=quantum-computing-plans) を参照してください。

このチャネルは、クラウドベースの開発環境をサポートしていません。 そのため、[QiskitとQiskitのRuntimeをインストールしてセットアップする](install#local)と[IBM Cloudを使用する設定](#cloud) が必要になります。

利用可能なプラン:

-   **スタンダードプラン** - 世界最高の量子システムで量子回路を実行し、あなたが使用する量子時間のみを支払います。

-   **Lite plan**: フリーシミュレータを使用して量子回路をデバッグして学びましょう。

## IBM Quantum Platformを使用するように設定

クラウドベースの IBM Quantum システムを使用するには、認証情報にアクセスする必要があります。 

1.  If you do not already have a user account, get one at the [IBM Quantum login page.](https://quantum.ibm.com/login)  Your user account is associated with one or more [instances](../run/instances) (in the form hub / group / project) that give access to IBM Quantum services. さらに、各アカウントに一意のトークンが割り当てられ、QiskitからIBM Quantumにアクセスすることができます。 このセクションの手順では、デフォルトのインスタンスを使用します。  特定のインスタンスを選択する手順については、format@@0(../run/instances#connect-instance)を参照してください。 

    <Admonition type="note">
      [IBM Quantum account page](https://quantum.ibm.com/account) のインスタンスセクションには、アクセスできるインスタンスが表示されます。
    </Admonition>

2.  [IBM Quantum account page,](https://quantum.ibm.com/account) から IBM Quantum トークンを取得し、Python を起動します。 例:

    ```shell
    python3
    ```

3.  IBM Cloud API キーと CRN で `QiskitRuntimeService` を呼び出してサービスに認証します。

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    service = QiskitRuntimeService(channel="ibm_quantum", token="<MY_IBM_QUANTUM_TOKEN>")

    ```

    または、オプションで `save_account()` メソッドを使用して、サービスを初期化する前に、後で簡単にアクセスできるように認証情報を保存します。

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    # Save an IBM Quantum account and set it as your default account.
    QiskitRuntimeService.save_account(channel="ibm_quantum", token="<MY_IBM_QUANTUM_TOKEN>", set_as_default=True)

    # Load saved credentials
    service = QiskitRuntimeService()
    ```

    -   ディスクに認証情報を保存する場合は、将来的に `QiskitRuntimeService()` を使用してアカウントを初期化できます。 `channel` パラメータは異なるアカウントの種類を区別します。 チャンネルごとに複数のアカウントを保存している場合は、`name`パラメータを使用して区別することを検討してください。 
    -   チャンネルごとに複数のアカウントを保存している場合は、`name`パラメータを使用して区別することを検討してください。 
    -   資格情報は、`$HOME/.qiskit/qiskit-ibm.json`に保存されます。  このファイルを手動で編集しないでください。 
    -   資格情報を保存しない場合は、新しいセッションを開始するたびにそれらを指定する必要があります。 
    -   `channel` パラメータは、異なるアカウントの種類を区別することができます。 アカウントを初期化する際には、IBM Quantum PlatformとIBM Cloudアカウントに資格情報を保存している場合に使用されるデフォルトのアカウントになります。

    <Admonition type="caution">
       アカウントの資格情報はプレーンテキストで保存されますので、信頼できるデバイスを使用している場合にのみ使用してください。
     </Admonition>

    <Admonition type="note">
     IBM Cloud は、別のチャンネルまたはアカウント名を指定しない場合に使用されるデフォルトのアカウントです。 
     </Admonition>

4.  セットアップをテストします。  サンプラーを使用して簡単な回路を実行して、環境が正しく設定されていることを確認します。

    ```python
     from qiskit import QuantumCircuit
     from qiskit_ibm_runtime import QiskitRuntimeService, Sampler

     # Create empty circuit
     example_circuit = QuantumCircuit(2)
     example_circuit.measure_all()

     # You'll need to specify the credentials when initializing QiskitRuntimeService, if they were not previously saved.
     service = QiskitRuntimeService()
     backend = service.backend("ibmq_qasm_simulator")
     job = Sampler(backend).run(example_circuit)
     print(f"job id: {job.job_id()}")
     result = job.result()
     print(result)
    ```

## IBM Cloud を使用するように設定

1.  Start Python. 例:

    ```shell
    python3
    ```

2.  まだお持ちでない場合は、[IBM Cloud Registration page](https://cloud.ibm.com/registration)からIBM Cloud アカウントを設定してください。

3.  必要に応じてサービスインスタンスを作成します。 [IBM Cloud Instances] \(https://cloud.ibm.com/quantum/instances) を開きます。 1つまたは複数のインスタンスが表示されている場合は、次のステップに進みます。 それ以外の場合は、**インスタンスを作成**をクリックします。 インスタンスを作成する際には、インスタンスに名前を付けてタグを付け、そのインスタンスのリソースグループを選択し、パフォーマンス戦略を選択できます。 次に、ページの右下にあるボックスにチェックを入れ、使用許諾契約に同意し、**作成**をクリックします。

    <Admonition type="note">
      組織の QisKit Runtime を設定する必要がある管理者の場合。 format@@0(https://cloud) を参照してください。 bm.com/docs/quantum-computing?topic=quantum-computing-quickstart-org).
    </Admonition>

    <Admonition type="note">
     パフォーマンス戦略を選択する場合、2つのオプションがあります。 IBM(デフォルト)の1つ、Q-CTRLのもう1つ。 IBMのパフォーマンス戦略では、[Qiskit Runtime](../api/qiskit-ibm-runtime)で利用可能なすべての標準オプションを活用でき、Q-CTRL戦略では自動化されたプリセットを使用できます。  Q-CTRLオプションの詳細については、format@@0(https://docs.q.ctrl.com/q-ctrl-embedded)をご覧ください。
    </Admonition>

4.  アクセスの資格情報を検索します。
    1.  API キーを見つけます。 format@@0(https://cloud.ibm.com/iam/apikeys)からAPIキーを表示または作成し、認証に使用できるように安全な場所にコピーします。
    2.  クラウドリソース名 (CRN) を検索します。 format@@0(https://cloud.ibm.com/quantum/instances)を開き、インスタンスをクリックします。 開いたページで、アイコンをクリックしてCRNをコピーします。 認証に使用できるように、安全な場所に保存してください。

5.  `QiskitRuntimeService` を保存した資格情報を使用して、またはIBM Cloud APIキーとCRNを使用してサービスに認証します。

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    service = QiskitRuntimeService(channel="ibm_cloud", token="<IBM Cloud API key>", instance="<IBM Cloud CRN>")
    ```

    <Admonition type="note">
      Q-CTRLのパフォーマンス管理を含めるためにステップ3でインスタンスを設定した場合。 `QiskitRuntimeService()`を初期化するときは、追加の引数`channel_strategy="q-ctrl"`を含める必要があります。 Q-CTRLのパフォーマンス管理戦略の詳細については、format@@0(https://docs.q-ctrl.com/q-ctrl-embedded)をご覧ください。
    </Admonition>

    必要に応じて、 `save_account()` メソッドを使用して、サービスを初期化する前に、認証情報を保存しておくことができます。

    ```python
     from qiskit_ibm_runtime import QiskitRuntimeService

     # Save account to disk. 
     QiskitRuntimeService.save_account(channel="ibm_cloud", token="<IBM Cloud API key>", instance="<IBM Cloud CRN>", name="<account-name>")

     # Load saved credentials
     service = QiskitRuntimeService(name="<account-name>")
    ```

    -   ディスクに認証情報を保存する場合は、将来的に `QiskitRuntimeService()` を使用してアカウントを初期化できます。 `channel` パラメータは異なるアカウントの種類を区別します。 アカウントを初期化しているとき。 IBM Cloud は、IBM Quantum Platform および IBM Cloud アカウントの両方に資格情報を保存している場合に使用されるデフォルトのアカウントです。
    -   チャンネルごとに複数のアカウントを保存している場合は、`name`パラメータを使用して区別することを検討してください。 
    -   資格情報は、`$HOME/.qiskit/qiskit-ibm.json`に保存されます。  このファイルを手動で編集しないでください。 
    -   資格情報を保存しない場合は、新しいセッションを開始するたびにそれらを指定する必要があります。 

    <Admonition type="caution">
         アカウントの資格情報はプレーンテキストで保存されますので、信頼できるデバイスを使用している場合にのみ使用してください。
     </Admonition>

6.  セットアップをテストします。  サンプラーを使用して簡単な回路を実行して、環境が正しく設定されていることを確認します。

````python
 from qiskit import QuantumCircuit
 from qiskit_ibm_runtime import QiskitRuntimeService, Sampler

 # Create empty circuit
 example_circuit = QuantumCircuit(2)
 example_circuit.measure_all()

 # You'll need to specify the credentials when initializing QiskitRuntimeService, if they were not previously saved.
 service = QiskitRuntimeService()
 backend = service.backend("ibmq_qasm_simulator")
 job = Sampler(backend).run(example_circuit)
 print(f"job id: {job.job_id()}")
 result = job.result()
 print(result)
 ```
````
