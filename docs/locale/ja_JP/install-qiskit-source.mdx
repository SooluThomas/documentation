# Qiskit をソースからインストール

ソースから Qiskit をインストールすることで、Python Package Index (PyPI) リポジトリでバージョンを使用する代わりに、現在の開発バージョンにアクセスすることができます。 これにより、Qiskit コードの最新バージョンをより効率的に検査し拡張することができます。 

## 新しい仮想環境の作成と有効化

まず、Qiskit用の新しい仮想環境を作成します。 

```python
  python3 -m venv QiskitDevenv
  ソース QiskitDevenv/bin/activate
```

## Rust コンパイラーをインストール

Qiskitをコンパイルするには、Rustコンパイラをシステムにインストールする必要があります。 Rust コンパイラーをインストールするには、クロスプラットフォームの Rust インストーラー [rustup](https://rustup.rs/) または [another install method.](https://forge.rust-lang.org/インフラストラクチャ/other-installation-methods.html) を使用してください。 

## Install Qiskit

Qiskitをインストールするには、以下の手順に従ってください:

1.  Qiskit リポジトリをクローンします。

```python
  git clone https://github.com/Qiskit/qiskit.git
```

2.  `qiskit`ディレクトリに変更します。 

```python
  cd qiskit
```

3.  (オプション)テストまたはリンティングチェックを実行する場合は、開発者要件をインストールします。

```python
  pip install -r requirements-dev.txt
```

4.  Install `qiskit`. 

-   **標準インストール**: 

    ```python
      pip install .
    ```

-   **編集可能なモード**: このモードでは、プロジェクトへのコードの変更がある場合、Qiskit を再インストールする必要はありません。

    ```python
      pip install -e .
    ```

    編集可能なモードでは、コンパイルされた拡張機能は最適化なしに _debug mode_ で構築されます。 これはコンパイルされたコードのランタイムパフォーマンスに影響します。 最適化が有効になってコンパイルされた拡張機能をビルドするには、_release mode_ でバイナリを再構築するには、次のコマンドを実行します。

    ```python
      python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    QiskitでRustコードを扱っている場合は、ローカルに変更を加えるたびに拡張コードを再構築する必要があります。 編集可能なモードでは、Rust 拡張は install コマンドが実行されたときにのみビルドされます。 ですから、Rust コードに加えたローカルの変更は、`build_rust` を使って拡張機能を再構築しない限り、インストールされたパッケージには反映されません(`--release` を使ってもかまいません)。 リリースモードまたはデバッグモードでビルドするかによって異なります)。
    </Admonition>
